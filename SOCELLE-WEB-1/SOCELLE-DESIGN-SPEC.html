01 / Creative Direction

10 Principles + 5 Rules

These are non-negotiable constraints that every component, page, and copy decision must pass through.

Design Principles

#PrincipleIn practice
01Intelligence leads. Marketplace follows.Every page opens with a signal, insight, or outcome — never a product grid.
02SOCELLE is the gallery wall.Our chrome is achromatic. Brand color, product photography, and marketing assets belong to the brands. We never compete.
03Data is the product. Design proves it.Animated metrics, count-ups, live tickers, and heatmaps are not decoration — they demonstrate that the platform is thinking.
04One dark panel per page.The void panel appears exactly once — as the primary hero or metrics anchor. It creates a cinematic contrast moment. More than one cheapens it.
05Glass surfaces need depth behind them.The atmospheric background gradients are not cosmetic. They are the substance that makes the glass blur visible. Never render glass over a flat surface.
06Type IS the design.On-brand moments are often just one large line of text with nothing competing. Give type space. 80px+ display at appropriate moments. Trust the scale.
07Every section answers three questions."What is it?" "Why should I care?" "Show me proof." If a section can't answer all three, it gets cut or combined.
08Motion communicates causality, not decoration.Animate to show what changed, why it changed, and what the user should do next. Never animate without purpose.
09Proof before claims.Any stat, headline, or value prop must be immediately accompanied by a mechanism that makes it credible — a number, a methodology note, a testimonial, a live data point.
10Premium restraint.If you're unsure whether to add something, remove it. Confidence is expressed through what's left out, not what's packed in.

Do / Don't Rules

Do
Use atmospheric radial gradient backgrounds — multiple overlapping light sources at different intensities create depth that makes glass visible and the page feel alive.
Don't
Use flat solid backgrounds behind glass elements. Glass over flat = opaque. The blur effect becomes invisible and the premium quality evaporates.
Do
Make signal colors (green, amber, red) exclusive to data context. They appear on chips, charts, badges — never as section backgrounds, button colors, or icon fills.
Don't
Use green for "success" UI states, amber for warnings, or red for errors in chrome. This bleeds the signal color system into UI chrome and breaks the rule that only data carries color.
Do
Write display headlines at 80-96px with weight 600 and -0.03em tracking. Let one sentence dominate a section completely. Negative space confirms the message.
Don't
Scale headlines below 32px for what are meant to be high-impact moments. "Professional grade intelligence" at 18px with 3 competing CTAs reads as a startup template, not a premium platform.
Do
Pair every animated metric or live count with a methodology disclosure — even a single line: "Based on aggregated data from 3,400+ operators." This converts data-feel into data-trust.
Don't
Assert "real-time" or "live" signals without defining what that means. If data updates daily, say daily. Trust is built by precision, not by ambiguity that implies more currency than exists.
Do
Confine gradient-art image placeholders to specific moody compositions with grain texture overlay. They should read as intentional atmospheric photography, not "we ran out of images."
Don't
Use stock photography of people in spas, white bottles on marble, or geometric shapes on gradients. These signals "generic SaaS" immediately. If real product photography isn't available, gradient art is better.

03 / Motion System

Single motion language.

All motion communicates causality or state change. Nothing animates for decoration. The system is fully specified — one implementation choice and one set of tokens governs the entire product.

Animation Engine: GSAP + ScrollTrigger

GSAP is the production choice. Reasons: timeline control for coordinated multi-element sequences, ScrollTrigger for scroll-driven narratives without IntersectionObserver polling, and performance parity with Framer Motion at lower bundle overhead for non-React contexts (public pages). In portal pages (React), use Framer Motion for component-level animations with GSAP for page-level scroll choreography.

The prototype in SOCELLE-REDESIGN-v2.html uses IntersectionObserver as a parity replacement — all behaviors map 1:1 to GSAP ScrollTrigger.

Hover Elevation System

All hover transitions use --t-std (280ms) with --ease. The rule is micro-elevation: a card lifts 3px and its shadow deepens. Nothing scales, warps, or changes color beyond the hover state.

Reduced Motion

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
  /* Reveal elements immediately — no translate, no fade */
  .reveal, .stagger > * {
    opacity: 1 !important;
    transform: none !important;
  }
  /* Stop ticker */
  .ticker-inner { animation: none !important; }
}

Scrubber, simulator, and expandable cards are all interactive-only (no animation) and fully accessible in reduced-motion mode.

<script>
const videoObs = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (!e.isIntersecting) return;
    const v = e.target;
    v.src = v.dataset.src;
    v.play().catch(() => {}); // catch autoplay policy
    videoObs.unobserve(v);
  });
}, { rootMargin: '200px' });

document.querySelectorAll('.lazy-video').forEach(v => videoObs.observe(v));
</script>
