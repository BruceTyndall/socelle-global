# Dev Workflow Scheduler

Schedules development tasks into automated pipelines. Instead of running audits, builds, and tests manually and hoping nothing gets skipped, this skill creates sequenced workflows that ensure the right checks run in the right order at the right time.

## Workflow Templates

### Pre-Merge Pipeline
```yaml
name: pre-merge
trigger: Before PR merge
steps:
  - smoke-test-suite        # 30s — quick sanity
  - build-gate              # 2min — tsc + vite build
  - secrets-scanner         # 15s — no leaked keys
  - rls-auditor             # 30s — RLS on all tables
  - doc-gate-enforcer       # 20s — governance compliance
  - proof-pack              # 10s — aggregate results
```

### Nightly Audit Pipeline
```yaml
name: nightly-audit
trigger: Daily at 2:00 AM
steps:
  - repo-auditor            # Full inventory
  - dependency-scanner      # npm audit + outdated
  - live-demo-detector      # Fake-live check
  - token-drift-scanner     # Design compliance
  - language-linter         # Copy quality
  - route-mapper            # Route completeness
  - proof-pack              # Bundle results
```

### Weekly Deep Audit
```yaml
name: weekly-deep
trigger: Sunday at 1:00 AM
steps:
  - repo-auditor
  - db-inspector
  - migration-validator
  - edge-fn-health
  - intelligence-module-checker
  - ai-service-menu-validator
  - entitlement-validator
  - performance-profiler
  - accessibility-checker
  - seo-audit
  - legal-compliance-checker
  - risk-gatekeeper
  - proof-pack
```

## Step 2: Build Execution Script

Generate a bash script that runs skills in sequence:

```bash
#!/bin/bash
# Auto-generated workflow: [name]
# Generated by dev-workflow-scheduler

set -e
RESULTS_DIR="docs/qa"
mkdir -p "$RESULTS_DIR"

echo "=== WORKFLOW: [name] ==="
echo "Started: $(date -u +%Y-%m-%dT%H:%M:%SZ)"

# Step 1: [skill-name]
echo "--- Running [skill-name] ---"
# [skill commands]

# Step N: proof-pack (always last)
echo "--- Running proof-pack ---"
# Aggregate all results

echo "Completed: $(date -u +%Y-%m-%dT%H:%M:%SZ)"
```

## Verification (Deterministic)
- **Command:** `find docs/qa -name "dev-workflow-scheduler*.json" | wc -l  # expect >= 1`
- **Pass criteria:** Command returns expected value (see comment); JSON report written to `docs/qa/dev-workflow-scheduler-YYYY-MM-DD.json`
- **Fail criteria:** Command returns unexpected value, report not generated, or any BLOCK-level issue found without resolution


## Stop Conditions
- **Hard stop:** Codebase does not compile (`npx tsc --noEmit` fails) — fix build before running this skill
- **Hard stop:** Target path does not exist or is empty — verify repo checkout
- **Hard stop:** Required command-line tool missing (grep, find, node) — install before proceeding
- **Soft stop (owner decision):** More than 50 issues found — present summary to owner for prioritization before attempting fixes
- **Soft stop (owner decision):** Findings contradict a command doc — escalate to owner; do not auto-fix governance
